⚙ Profile-Based Database Selection

Core mechanism: SpringApplicationContextInitializer controls which database backend is active. It maps Spring profiles to service tags, validates that only one profile is active, and disables irrelevant auto-configurations.

Startup Flow

1. Validate single profile
→
2. Detect CF services (if on CF)
→
3. Activate matching profile
→
4. Exclude unused auto-configs

Auto-Configuration Exclusion Matrix

Active ProfileExcluded Auto-Configurations
redis DataSourceAutoConfiguration, HibernateJpaAutoConfiguration, MongoAutoConfiguration, MongoDataAutoConfiguration
mongodb DataSourceAutoConfiguration, HibernateJpaAutoConfiguration, RedisAutoConfiguration, RedisRepositoriesAutoConfiguration
JPA (default) MongoAutoConfiguration, MongoDataAutoConfiguration, RedisAutoConfiguration, RedisRepositoriesAutoConfiguration
Constraint: Only one database profile may be active at a time. Activating multiple profiles throws an IllegalStateException.

☰ Persistence Layer

Album Entity

@Entity
public class Album {
    @Id
    @GeneratedValue(generator = "randomId")
    @GenericGenerator(name = "randomId",
        strategy = "...RandomIdGenerator")
    @Column(length = 40)
    private String id;

    private String title;
    private String artist;
    private String releaseYear;
    private String genre;
    private int trackCount;
    private String albumId;
}

Uses RandomIdGenerator (UUID) for cross-backend ID compatibility.

Repository Activation

// JPA — active when NOT mongo or redis
@Repository
@Profile({"!mongodb", "!redis"})
public interface JpaAlbumRepository
    extends JpaRepository<Album, String> {}

// MongoDB — only with mongodb profile
@Repository
@Profile("mongodb")
public interface MongoAlbumRepository
    extends MongoRepository<Album, String> {}

// Redis — custom CrudRepository impl
@Repository
@Profile("redis")
public class RedisAlbumRepository
    implements CrudRepository<Album, String> {
    // Uses RedisTemplate HashOperations
    // All albums stored in hash "albums"
}
Why Redis is special: Redis doesn't have a Spring Data Repository abstraction, so RedisAlbumRepository manually implements CrudRepository using HashOperations<String, String, Album>. Albums are stored as a Redis hash under the key "albums", with each album's ID as the hash field. A custom RedisConfig sets up Jackson JSON serialization.

★ Data Seeding — AlbumRepositoryPopulator

Listens for ApplicationReadyEvent and populates the repository from albums.json when it's empty.

App starts
→
ApplicationReadyEvent fires
→
Check count() == 0
→
Deserialize albums.json
→
Save all to active repository
Decoupled lookup: Uses BeanFactoryUtils.beanOfTypeIncludingAncestors() to dynamically discover whichever CrudRepository bean is active — no hard dependency on a specific implementation.

▶ Application Entry Point

@SpringBootApplication
public class Application {
    public static void main(String[] args) {
        new SpringApplicationBuilder(Application.class)
            .initializers(new SpringApplicationContextInitializer())  // profile setup
            .listeners(new AlbumRepositoryPopulator())          // data seeding
            .application()
            .run(args);
    }
}

The initializer runs before the Spring context is created (profile & auto-config setup), while the populator runs after the context is ready (data seeding).
